# Getting Started

This section is the shortest path from zero context to a trustworthy first result.

## Recommended path

1. [Install](install.md)
2. [Quickstart](quickstart.md)
3. [First Local Run](first-local-run.md)
4. [First CI Setup](first-ci-setup.md)

## What success looks like

After completing this section, you should be able to:

- build or install the Meridian binary
- run `validate` and understand its output
- run `check` in `safe` mode and inspect the resulting artifacts
- connect the same workflow to GitHub Actions for pull request feedback

## Before you continue

If you are evaluating Meridian for a team, it is worth reading [Evidence Model](../core-concepts/evidence-model.md) after the quickstart. That page explains what Meridian's results mean and where their trust boundary ends.
